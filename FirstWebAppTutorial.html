  • VSCode: A widely using coding environment with AI assistance, perfect for beginners. VSCode has over 14 million monthly active users.
  • GitHub: Your collaborative playground where you can keep your code and collaborate with others. GitHub hosts over 100 million repositories and has more than 83 million developers.

2. Running a web application

Project Description

This project is a web application designed to track material and labor costs for building a house.

Purpose

The web application helps users view and select materials for different parts of a house. It aims to calculate the total cost based on the selected materials.

Technologies Used

  • HTML (index.html): Used to create the structure of the web pages.
  • JavaScript (script.js): Used to add interactivity to the web pages, such as loading data and updating the display.
  • Python (houseparts.py): Used on the server side to handle requests and process data.
  • Excel (houseparts.xlsx): The data about house parts and materials is stored in an Excel file.
  • Flask: A lightweight web framework for Python that is used to create the server and API endpoints.
  • Pandas: A Python library used to read and manipulate data from an Excel file.
  • CORS: A Flask extension that allows the web application to request resources from the server.

How It Works

  1. The user opens a web page (index.html) that displays a table for house parts.
  2. When the user clicks a button to load data, the JavaScript code (script.js) sends a request to a Flask server (note: the server needs to be started by running the python script in debug mode).
  3. The Flask server reads the data from an Excel file using the Pandas library and sends it back to the web page as JSON (you can see the JSON by going to localhost:5000/get_house_parts).
  4. The web page then displays the data in a table, allowing the user to select materials for each house part.

Overall, this project combines front-end web technologies (HTML, JavaScript) with back-end server technologies (Python, Flask) to provide a seamless experience for tracking and selecting building materials.

Step-by-Step Setup:
  1. Open the repo in VSCode
    • Open the folder for the repo in VSCode using the Explorer icon in the left nav.
    • Open the python script (houseparts.py) that contains the code to start the Flask server
  2. Start the Flask server
    • Open a terminal (Menu->View->Terminal)
    • Run the python script (houseparts.py) in debug mode (using the run+bug icon on the left nav). Observe that you will get errors in the terminal window because you haven't installed the right dependencies
  3. Debug possible issues starting the server

    4. There are lots of sources of errors that I ran into... These are some ways to debug.

    • Errors in code: Try using Copilot to fix the errors, by selecting the code that is wrong, right click, and selecting the "Fix" option from the Copilot menu. Since I did not add a step above to install the libraries used in houseparts.py, you should run into errors with missing dependencies.
    • Install the dependencies 
      pip install flask pandas flask_cors openpyxl
    • Try running the program again (using the button in the left nav). This should start a flask server at the following address: http://127.0.0.1:5000
    • Errors starting the server: You may get an error in the terminal saying "ConnectionRefusedError: [WinError 10061] No connection could be made because the target machine actively refused it". This is likely because you ran the program from the terminal command and it is not running in debug mode
    • If you are successful, the terminal should say

      WARNING: This is a development server. Do not use it in a production deployment. Use a production WSGI server instead.

      * Running on http://127.0.0.1:5000

  4. Check the REST endpoint
    • Open a browser and navigate to http://127.0.0.1:5000/get_house_parts. This is where the Flask server hosts the function that the JavaScript will call.
    • You should see the following

      [{"Material 1":"Shingles","Material 2":"Metal","Material 3":"Tiles","Part":"Roof"},{"Material 1":"Brick","Material 2":"Concrete","Material 3":"Wood","Part":"Walls"},{"Material 1":"Concrete","Material 2":"Stone","Material 3":"Pavers","Part":"Foundation"}]

  5. Open the web page and load the data
    • Open the index.html file that is in the repo in the browser
    • Click the "Load Data" button to load the data from the Flask server
    • Observe that the data is displayed in the table
  6. Debugging the web app
    • It is always smart to open the dev tools before you load a page. Use F12 to open the sidepane.
    • You can click on the terminal icon to see messages and errors
